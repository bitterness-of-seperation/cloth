# 匿名登录功能说明

## 修改原因

Supabase 免费版有邮件发送频率限制（每小时 3-4 封），导致用户注册时经常遇到 "email rate limit exceeded" 错误。

为了提供更好的用户体验，我们改用匿名登录方式。

## 功能特点

### 优点
✅ 无需邮箱验证，即刻使用  
✅ 无邮件发送限制  
✅ 注册流程简化，只需输入用户名  
✅ 降低用户使用门槛  

### 注意事项
⚠️ 匿名账号数据保存在浏览器本地  
⚠️ 清除浏览器数据会丢失账号信息  
⚠️ 每次登录创建新的匿名会话  
⚠️ 不支持跨设备同步数据  

## 使用方式

### 注册
1. 访问注册页面
2. 输入用户名（2-20 个字符）
3. 点击"立即开始"
4. 自动登录并跳转到试穿页面

### 登录
1. 访问登录页面
2. 输入用户名
3. 点击"登录"
4. 创建新会话并跳转

## 技术实现

使用 Supabase 的匿名认证功能：

```typescript
// 注册/登录
const { data, error } = await supabase.auth.signInAnonymously({
  options: {
    data: { username: username.trim() }
  }
});
```

## Supabase 配置

需要在 Supabase Dashboard 中启用匿名登录：

1. 登录 Supabase Dashboard
2. 进入项目设置
3. 导航到 `Authentication` → `Providers`
4. 找到 `Anonymous Sign-ins`
5. 启用该选项

## 数据存储

### 用户数据
- 用户名存储在 `user_metadata.username`
- 会话信息存储在浏览器 localStorage
- Supabase 自动管理匿名用户 ID

### 试穿记录
- 关联到匿名用户 ID
- 存储在 Supabase 数据库
- 清除浏览器数据后无法恢复

## 升级到正式账号（可选）

如果未来需要支持跨设备同步，可以添加"绑定邮箱"功能：

```typescript
// 将匿名账号升级为正式账号
const { data, error } = await supabase.auth.updateUser({
  email: 'user@example.com',
  password: 'new-password'
});
```

## 对比

| 功能 | 邮箱注册 | 匿名登录 |
|------|---------|---------|
| 注册速度 | 慢（需验证邮箱） | 快（即刻使用） |
| 邮件限制 | 有限制 | 无限制 |
| 跨设备同步 | 支持 | 不支持 |
| 数据持久化 | 永久 | 依赖浏览器 |
| 找回密码 | 支持 | 不支持 |
| 用户体验 | 一般 | 优秀 |

## 适用场景

匿名登录适合：
- 快速体验产品
- 不需要跨设备同步
- 临时使用场景
- 降低注册门槛

不适合：
- 需要长期保存数据
- 多设备使用
- 企业级应用

## 安全性

- Supabase 为每个匿名用户生成唯一 ID
- 会话令牌加密存储
- 支持 Row Level Security (RLS) 策略
- 匿名用户只能访问自己的数据

## 回滚方案

如需恢复邮箱注册，只需：
1. 恢复 `login/page.tsx` 和 `register/page.tsx` 的原始代码
2. 在 Supabase 中禁用匿名登录
3. 启用邮箱验证

原始代码已保存在 Git 历史中，可随时回滚。
