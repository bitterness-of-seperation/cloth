# 虚拟试穿 App — Ubuntu 服务器部署文档

## 前提条件

- Ubuntu 20.04 / 22.04 服务器
- 有公网 IP（假设为 `YOUR_SERVER_IP`）
- 已开放端口：`3000`（前端）、`8000`（后端）

---

## 一、服务器初始化

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y git curl wget unzip build-essential

# 安装 Python 3.11
sudo apt install -y python3.11 python3.11-venv python3-pip

# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 验证版本
python3.11 --version   # Python 3.11.x
node --version         # v20.x.x
npm --version          # 10.x.x
```

---

## 二、上传代码

**方式A：Git（推荐）**
```bash
cd ~
git clone https://github.com/你的用户名/虚拟试穿app.git app
cd app
```

**方式B：scp 直接上传（Windows 本地执行）**
```powershell
# 在本地 PowerShell 执行
scp -r "E:\虚拟试穿app" user@YOUR_SERVER_IP:~/app
```

## 三、部署后端

```bash
cd ~/app/backend

# 创建虚拟环境
python3.11 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env .env.bak   # 备份
nano .env
```

`.env` 内容（按实际填写）：
```env
SUPABASE_URL=https://ftnwpvqmksaolgwuljbe.supabase.co
SUPABASE_SERVICE_KEY=your_supabase_service_key_here

HF_TOKEN=your_hf_token_here

RAPIDAPI_KEY=your_rapidapi_key_here
```

**测试后端能否启动：**
```bash
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000
# 看到 "Application startup complete." 后 Ctrl+C
```

**用 systemd 守护进程运行：**
```bash
sudo nano /etc/systemd/system/tryon-backend.service
```

写入以下内容（注意替换用户名）：
```ini
[Unit]
Description=虚拟试穿 FastAPI 后端
After=network.target

[Service]
User=YOUR_USERNAME
WorkingDirectory=/home/YOUR_USERNAME/app/backend
Environment="PYTHONIOENCODING=utf-8"
ExecStart=/home/YOUR_USERNAME/app/backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
# 启动并设置开机自启
sudo systemctl daemon-reload
sudo systemctl enable tryon-backend
sudo systemctl start tryon-backend

# 查看状态
sudo systemctl status tryon-backend

# 查看日志
sudo journalctl -u tryon-backend -f
```

## 六、常用运维命令

```bash
# 重启服务
sudo systemctl restart tryon-backend
sudo systemctl restart tryon-frontend

# 停止服务
sudo systemctl stop tryon-backend
sudo systemctl stop tryon-frontend

# 实时查看后端日志（含试穿策略）
sudo journalctl -u tryon-backend -f

# 更新代码后重新部署
cd ~/app
git pull                          # 拉取最新代码

# 重新构建前端
cd frontend
npm install && npm run build
sudo systemctl restart tryon-frontend

# 重启后端（依赖不变则无需重装）
sudo systemctl restart tryon-backend
```

## 七、开放防火墙端口

```bash
# Ubuntu UFW
sudo ufw allow 3000/tcp
sudo ufw allow 8000/tcp
sudo ufw status
```

如果是云服务器（阿里云/腾讯云/AWS），还需要在控制台安全组里开放 3000 和 8000 端口。

## 八、故障排查

| 现象 | 排查命令 | 可能原因 |
|------|---------|---------|
| 前端打不开 | `sudo journalctl -u tryon-frontend -f` | build 失败 / 端口被占 |
| 后端 502 | `sudo journalctl -u tryon-backend -f` | Python 依赖缺失 / .env 未配置 |
| 试穿一直 fallback | 后端日志看 `[策略1/3]` 失败原因 | HF_TOKEN 无效 / 网络不通 HF |
| 图片上传失败 | 检查 Supabase Storage bucket 权限 | bucket 未设为公开 |
| CORS 错误 | 检查 `backend/.env` 的 `CORS_ORIGINS` | 前端域名未加入白名单 |

**CORS 配置（如果前端域名不是 IP:3000）：**

在 `backend/.env` 添加：
```env
CORS_ORIGINS=http://YOUR_SERVER_IP:3000,https://你的域名
```
